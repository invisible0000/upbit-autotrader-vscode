# 📋 업비트 자동매매 프로젝트 태스크 관리 시스템

## 🎯 목적
- **주요 목표**: 체계적이고 추적 가능한 태스크 관리를 통한 프로젝트 효율성 극대화
- **보조 목표**: LLM 에이전트 간 원활한 작업 승계 및 진행 상황 투명성 확보

## 📁 폴더 구조
```
📁 tasks/
├── 📄 README.md                    # 태스크 관리 시스템 가이드 (이 문서)
├── 📁 active/                      # 🔥 현재 진행중인 태스크들
├── 📁 completed/                   # ✅ 완료된 태스크들
├── 📁 planned/                     # 📅 계획된 태스크들
└── 📁 archived/                    # 🗂️ 보관된 태스크들
```

## 🏷️ 네이밍 규칙
**태스크 파일명 형식**:
```
TASK_{YYYYMMDD}_{순번}-{카테고리}_{간단한_설명}.md

예시:
- TASK_20250817_01-db_schema_analysis.md
- TASK_20250817_02-ui_trigger_builder_fix.md
- TASK_20250818_01-backtest_engine_improvement.md
```

**태스크 ID 체계**:
- `YYYYMMDD`: 8자리 날짜 (2025년 8월 17일 = 20250817)
- `순번`: 2자리 숫자 (01, 02, 03...)
- `카테고리`: db, ui, backend, testing, docs 등
- `설명`: 밑줄(_)로 구분된 간단한 설명

## 🎯 태스크 상태 관리 시스템

### 🔥 진행중 (Active)
- **목적**: 현재 작업하고 있는 태스크들
- **사용법**: 매일 아침 여기를 확인하여 오늘 할 일을 파악
- **원칙**: 동시 진행 태스크는 최대 3개로 제한

### ✅ 완료 (Completed)
- **목적**: 완료된 태스크들의 결과물과 소요시간 기록
- **활용**: 성과 측정 및 학습 자료, 유사 작업 시 참고
- **보관**: 완료일 기준 6개월 후 archived로 이동

### 📅 계획 (Planned)
- **목적**: 앞으로 해야 할 태스크들의 우선순위와 일정 관리
- **관리**: 전체 프로젝트 로드맵 및 마일스톤 추적
- **업데이트**: 주간 단위로 우선순위 재검토

### 🗂️ 보관 (Archived)
- **목적**: 취소되거나 보류된 태스크들
- **기준**: 장기간 미진행 또는 요구사항 변경으로 인한 중단
- **활용**: 나중에 재검토할 가능성이 있는 작업들

3. **파일 이동**: `active/`에서 `completed/`로 이동
4. **다음 작업 준비**: 후속 태스크 생성 또는 업데이트
