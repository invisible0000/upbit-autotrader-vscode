# 통합 테스트(E2E, 성능, 스트레스) 시나리오 기획

작성일: 2025-07-19

---

## 1. 실시간 거래 시나리오
- 실시간 데이터 수신 → 조건 충족 시 자동 주문 → 포지션/상태 반영까지 전체 플로우 검증
- 단계:
  1. 웹소켓으로 실시간 가격 데이터 수신(모의 데이터 사용 가능)
  2. 거래 조건(예: 가격 급등/급락) 충족 시 주문 생성/전송
  3. 주문 체결 후 포지션/상태/PNL 반영
  4. 알림/로그 기록 확인

## 2. 다중 코인/다중 주문 시나리오
- 여러 코인에 대해 동시에 주문/포지션/상태 관리가 정상 동작하는지 검증
- 단계:
  1. 여러 코인(KRW-BTC, KRW-ETH 등) 거래 설정
  2. 각 코인별 주문 생성/전송/취소/수정
  3. 포지션/상태/PNL 개별 관리 및 전체 합산 검증

## 3. 주문 실패/오류 처리 시나리오
- 주문 실패(잔고 부족, 네트워크 오류 등) 상황에서 시스템이 안전하게 동작하는지 검증
- 단계:
  1. 주문 전송 시 실패 상황(잔고 부족, 네트워크 장애 등) 모의
  2. 오류 발생 시 적절한 예외 처리/알림/상태 복구 확인

## 4. 성능/스트레스 테스트 시나리오
- 대량 주문/데이터 처리 시 시스템의 속도와 안정성 검증
- 단계:
  1. 1000건 이상의 주문/데이터를 반복 처리
  2. 처리 시간, 메모리/CPU 사용량 측정
  3. 과부하 상황에서 오류/지연/데이터 손실 없는지 확인

## 5. UI 연동 통합 시나리오
- 실제 GUI(데스크탑/웹)에서 거래/모니터링/알림 등 모든 기능이 정상 동작하는지 검증
- 단계:
  1. UI에서 거래 설정/주문/포지션/알림 등 주요 기능 실행
  2. 백엔드와의 연동 결과(상태, 로그, 알림 등) 실시간 확인
